# 공개 문서 정책

## 목적

이 저장소는 LocalAssistant가 어떤 문제를 해결했고, 어떤 변경을 거쳤으며, 그 결과를 어떻게 검증했는지 설명합니다. 구현 자체를 배포하는 소스 저장소가 아닙니다.

리포트 원문은 비공개 소스 저장소에도 함께 유지합니다. 공개 GitHub 저장소로의 반영은 원문 중 아래 기준을 통과한 문서만 대상으로 하며, 동기화 방향은 비공개 소스 저장소에서 공개 문서 저장소로 한정합니다.

## 게시할 수 있는 내용

- 구현한 기능의 목적과 사용자 관점 동작
- 제품 목표, 핵심 주제, 성공 기준과 의도적으로 제외한 범위
- 익명화된 진행 과정, 시행착오, 검토한 대안과 결정 근거
- 변경 전 문제와 변경 후 결과
- 익명화된 재현 조건과 평가 기준
- 테스트 범위, 총 개수, 통과·실패·오류·건너뜀 건수와 관찰된 한계
- 보안·개인정보 보호 원칙과 설계 결정
- 실제 운영에서 확인된 일반화 가능한 교훈

## 게시하지 않는 내용

- 제품 소스, 테스트 fixture와 실행 가능한 도구
- 개인별 Adapter의 구체적인 스키마와 라우팅 규칙
- 실제 사용자 질문, 가족 정보와 원본 수집 자료
- 호스트 이름, 계정, 로컬 경로, 포트 구성과 인증 정보
- 비공개 저장소의 커밋 차이 또는 소스 전문

## 결과 작성 기준

평가 문서는 평가일, 대상 버전 또는 변경 단위, 환경의 비식별 설명, 방법, 테스트 종류별 총 개수·통과·실패·오류·건너뜀 숫자, 결과, 한계와 결론을 포함합니다. 일부만 실행했다면 부분 테스트로 표시하고, 미실행이면 이전 숫자를 복사하지 않고 이유를 적습니다. 결과를 재현하는 데 내부 정보가 필요하면 원문 대신 익명화된 조건과 집계값만 공개합니다.

## 자동 차단

저장소 검사는 Markdown과 제한된 문서 첨부 형식만 허용합니다. 허용되지 않은 파일이 추적되면 변경 검사가 실패합니다. 자동 검사는 보조 장치이며, 문서 본문의 민감 정보는 작성자가 별도로 검토해야 합니다.
