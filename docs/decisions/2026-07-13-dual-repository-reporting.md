# 설계 결정: 소스와 공개 리포트 저장소 분리

- 날짜: 2026-07-13
- 상태: 채택

## 배경

제품 소스와 내부 운영 정보는 비공개로 유지하면서 실제 개발 과정, 수정 결과와 평가를 별도 GitHub 저장소에서 공유할 필요가 있었습니다.

## 고려한 대안

- 소스 저장소 전체를 공개 mirror로 운영하는 방식은 소스와 개인화된 운영 정보가 노출될 수 있어 제외했습니다.
- 공개 문서만 별도로 작성하는 방식은 공개 범위를 명확히 통제할 수 있지만, 원문 리포트와 공개본을 구분해 관리해야 합니다.

## 결정

개발 리포트 원문은 비공개 소스 저장소에도 함께 보존합니다. GitHub 저장소에는 공개 정책을 통과한 개발 기록, 변경 설명, 결과 분석과 평가 문서만 단방향으로 게시합니다.

일반적인 `커밋`과 `푸시` 요청은 비공개 소스 저장소를 대상으로 합니다. `GitHub`를 명시한 커밋과 푸시 요청만 공개 문서 저장소를 대상으로 하며, 두 저장소의 작업은 서로 독립적으로 수행합니다.

## 결과

소스 저장소는 기존 원격과 이력을 그대로 유지합니다. 공개 저장소는 문서 형식 허용 목록과 변경 검사를 사용해 소스 파일 유입을 차단합니다.

## 재검토 조건

공개 가능한 소스 배포를 별도로 결정하거나, 문서 공개 과정에서 반복적인 누락·민감 정보 노출 문제가 확인되면 게시 절차와 자동 검사를 재검토합니다.
