# 설계 결정: 개인 기반을 조직 환경으로 확장 가능한 공통 기반으로 다룬다

- 날짜: 2026-07-13
- 상태: 채택

## 배경

Local Assistant는 개인의 실제 생활 질문과 사용자 소유 자료로 제품 계약을 검증하고 있다. 이 출발점이 개인 전용 기능의 누적을 뜻하는지, 또는 충분히 다듬었을 때 조직에서도 재사용할 수 있는 기반을 뜻하는지 제품 경계를 명확히 할 필요가 있었다.

## 고려한 대안

- 개인용과 기업용 핵심 엔진을 별도로 만든다. 환경별 최적화는 쉽지만 기억·도구·권한 계약이 중복되고 두 제품이 서로 다른 방향으로 갈 위험이 있다.
- 기업 요구를 예상해 대규모 SaaS와 멀티테넌시를 먼저 만든다. 미래 확장을 앞당길 수 있지만 현재 실사용으로 확인되지 않은 복잡성과 운영 부담을 만든다.
- 개인 실증에서 공통 계약을 먼저 검증하고 조직별 차이는 Adapter와 통제 계층으로 흡수한다. 확장 시점은 늦을 수 있지만 현재 문제 해결과 장기 재사용성을 함께 유지할 수 있다.

## 결정

개인과 기업 환경에 Model Adapter, Data Adapter, Memory Lifecycle, 개인정보 보호 회귀 평가, 출처 보존, 최소 권한과 모델 독립성 원칙을 공통으로 적용한다. 개인 환경에는 개인 자료와 생활 도구 Adapter를, 조직 환경에는 사내 문서·DB·업무 도구 Adapter와 역할별 권한·감사·보존 정책을 연결한다.

현재는 개인 실사용으로 이 계약을 검증한다. 기업 적용 가능성만으로 대규모 SaaS, 다회사 멀티테넌시 또는 아직 필요가 확인되지 않은 관리 기능을 선행 구현하지 않는다.

## 결과

기업 확장은 별도 핵심 제품을 다시 만드는 일이 아니라 조직용 연결부와 통제 계층을 추가하는 장기 방향이 된다. 현재 구현 범위는 그대로 개인 실증이며, 이 문서화 작업에서 기업 기능을 새로 구현하거나 검증한 것은 없다.

## 재검토 조건

조직용 합성 데이터 원천을 기존 Data Adapter 계약으로 연결할 수 없거나, 역할별 권한·감사 요구가 개인 환경의 공통 계약을 훼손하거나, 실제 기업 파일럿에서 별도 핵심 엔진이 필요하다는 반복 가능한 근거가 확인되면 이 결정을 재검토한다.
