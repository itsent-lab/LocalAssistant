# 평가 기록: 목표 대비 구조 간극과 변경 동결

- 평가일: 2026-07-14
- 대상: 사용자 소유 개인 비서의 목표와 현재 운영 프로토타입
- 결과: 부분 통과
- 코드 변경: 없음

## 평가 목적

LocalAssistant의 다음 개발이 기능 수를 늘리는 방향이 아니라 다음 핵심 목표에 직접 기여하는지 점검했다.

- 기기·모델·공급자가 바뀌어도 기억과 사용법이 이어진다.
- 자연어로 허용된 자료를 찾고 자료 부재를 추측으로 채우지 않는다.
- 사용자 교정 뒤 유사 실패가 줄어든다.
- 사용할수록 사용자의 코드·설정·질문 수정 부담이 줄어든다.

이번 평가는 무엇을 구현할지 결정하는 것뿐 아니라 여러 미커밋 개발 단위 위에 구조 변경을 더하는 것이 안전한지도 함께 판단했다.

## 방법

비공개 소스 저장소에서 다음 범위를 읽기 전용으로 대조했다.

1. 제품 목표와 현재 검증 수준
2. 실제 대화 실행 경로의 Model·Data Adapter, Tool, Context와 평가 경계
3. 전체 Python 회귀 테스트
4. 원문 질문을 제외한 자동 평가 결과의 집계
5. 파일 책임 상한과 현재 작업 트리의 변경 상태

개인 질문, 개인 데이터 원문, 로컬 경로, 비공개 스키마와 인증 정보는 평가 문서에 포함하지 않았다.

## 결과

### 현재 확인된 기반

- 모델 요청 형식과 로컬 공급자 fallback을 분리하는 Adapter 경계가 있다.
- 읽기 전용 Data Adapter 계약과 개인정보 없는 합성 원천 회귀가 있다.
- 대화, 기억, 라우팅 교정과 Turn Trace가 모델 가중치 밖에 저장된다.
- Markdown Wiki를 사람이 읽고 수정할 수 있는 canonical 원본으로 유지한다.
- 현재 작업 트리의 전체 Python 회귀가 통과했다.

이는 현재 코드 계약의 비회귀 근거다. 다른 사용자의 독립 사용성, 장기 기억 정확성, 모델·기기 이전 뒤 연속성과 제품 성공률을 증명하지는 않는다.

### 자동 평가의 의미 제한

원문을 제외한 로컬 집계는 다음과 같았다.

- 전체 평가: 1,155건
- 자동 `passed`: 1,149건
- `needs_improvement`: 6건
- 감지된 실패 범주: 근거 부족 5건, 처리되지 않은 오류 1건

현재 자동 평가는 알려진 오류·정정·근거 부족 신호를 찾지 못하면 기본적으로 `passed`를 기록한다. 따라서 이 집계는 99%대 자연어 과업 성공률이 아니다. `passed`의 실제 의미는 `감지된 실패 신호 없음`에 가깝고, 사실 정확성·유용성과 사용자 만족은 확인하지 못했다.

성공을 과장하지 않기 위해 다음 상태를 분리해야 한다.

- 평가하지 못함 또는 알려진 실패 없음
- 명시적 사용자 확인
- 사용자 정정·거절
- 결정적 시스템 영수증으로 확인한 실행
- 자료·근거 부족
- 처리 오류

사용자의 무응답과 창 닫기를 긍정 근거로 사용하지 않는다. 선택형 피드백은 가능하지만 모든 답변을 사용자가 평가하도록 요구해서도 안 된다.

### 공통 Adapter가 실제 운영 경로의 단일 기준이 아님

공통 Data Adapter와 합성 두 번째 원천은 계약·회귀 수준에서 확인됐지만 실제 대화 운영 경로는 여전히 개인 도구별 Context와 연결부를 별도로 조립한다.

Tool 이름, Interpreter 허용 목록, 근거 capability, 자연어 라우팅, 권한과 Context 우선순위도 여러 경계에 분산돼 있다. 새 도구를 연결할 때 한 경계만 수정하면 선택·권한·실행·출처가 어긋날 수 있다.

새로운 범용 Agent 프레임워크를 선행할 필요는 없다. 두 번째 Data Adapter를 실제 대화 경로에 연결할 때 필요한 최소 범위에서 기존 운영 Tool 계약이 다음 정보를 함께 제공하도록 정리하는 것이 적절하다.

- 이름과 사용자 설명
- 읽기·쓰기 권한과 지원 동작
- 개인정보·외부 전송 범위
- 제공 가능한 근거와 freshness
- Context source와 우선순위
- 실제 실행 Adapter

Context Builder는 개인 도구별 고정 인자보다 공통 Tool 결과 목록을 소비하고, 개인 표현·사이트 파서는 환경별 Adapter에서 주입하는 방향이 목표에 더 가깝다.

### 남은 핵심 검증

다음 항목은 아직 제품 수준으로 확인되지 않았다.

1. 특정 로컬 자료 요청에서 불필요한 공개 Web 검색이 발생하지 않는가
2. 최신 예보 질문의 evidence scope가 실제 근거 범위와 일치하는가
3. 새 Data Adapter가 개인 전용 코드를 수정하지 않고 실제 대화 종단 경로에 연결되는가
4. 기억·교정·권한·평가 자산을 별도 환경으로 복원할 수 있는가
5. 모델 공급자를 바꾼 뒤에도 Tool 선택, 자료 부재와 기억 출처의 의미가 유지되는가
6. 설치·진단·백업·복구를 개발자 도움 없이 수행할 수 있는가

## 우선순위와 결정

권장 순서는 다음과 같다.

1. 자동 평가 상태의 의미를 바로잡고 비식별 기준선을 만든다.
2. 알려진 Query Interpreter·evidence gate 결함을 합성 회귀로 고친다.
3. 두 번째 Data Adapter를 실제 대화에 연결하면서 필요한 최소 Tool 계약 정리를 함께 수행한다.
4. 백업·복원과 모델 공급자 교체 검증을 추가한다.
5. 사용자 개입 감소를 측정할 수 있을 때 제한된 파일럿을 검토한다.

모바일 Connector, 범용 Agent·MCP, 자동 Cloud fallback, 조직 멀티테넌시와 감시 기능 확대는 현재 핵심 간극보다 먼저 구현하지 않는다.

현재 작업 트리에 서로 다른 개발 단위의 미커밋 변경이 남아 있으므로 이 평가 직후 새 구조 수정을 시작하지 않기로 했다. 다음 구현 전에 변경 경계, 누락 파일, 공개 제외 대상과 복구 가능성을 먼저 검토한다. 현재 상태를 고정하는 커밋도 명시적 요청과 검토 후에만 생성한다.

### 미실행

- 이 평가 작업에서 macOS 앱 빌드·서명은 실행하지 않았다.
- 실행 앱 재설치, 프로세스·health 확인은 실행하지 않았다.
- 실제 개인 자료 변경, 외부 알림과 실제 모델 질문은 실행하지 않았다.

미실행 이유는 실행 코드 변경 없이 분석과 문서화만 수행했기 때문이다.

## 한계

- 전체 회귀 통과는 비공개 소스에서 실행한 결과이며 공개 CI에서 독립 재현할 수 없다.
- 자동 평가 집계는 성공률이 아니라 현재 분류 편향을 확인하는 자료로만 사용했다.
- 장기 기억 정확성, 다른 사용자 독립 사용성과 기기 이전은 미검증이다.
- 평가 중 작업 트리가 달라졌으므로 후속 작업은 당시 파일 목록을 현재 상태로 가정하지 않고 실제 저장소 상태를 다시 확인해야 한다.

## 결론

현재 LocalAssistant는 핵심 계약을 V1~V3 수준에서 검증하는 운영 프로토타입이라는 기존 평가가 유지된다. 다음 진전은 기능 추가량이 아니라 성공·미확인의 정직한 구분, 실제 운영 Data Adapter 일반화, 근거 계약과 기억 이식성 검증에서 나와야 한다.

동시에 현재 변경 위에 새 구조 수정을 즉시 겹치지 않는 것이 안전하다. 이번 평가의 완료 결과는 구현이 아니라 검증 가능한 다음 순서와 변경 동결 조건을 확정한 것이다.
