# 평가 기록: 문서 전용 정책

- 평가일: 2026-07-13
- 대상: 문서 저장소 초기 구조
- 결과: 통과

## 평가 목적

소스 파일이 실수로 공개 저장소에 포함되는 것을 저장소 규칙으로 차단할 수 있는지 확인했습니다.

## 방법

추적 파일 목록을 기준으로 허용 확장자와 정확히 일치하는 관리 파일만 통과시키는 검사를 사용했습니다. 허용되는 Markdown 문서와 이미지·PDF 첨부 형식, 문서 전용 검사 파일을 구분했습니다.

## 결과

초기 저장소의 모든 파일이 허용 범위에 포함되었습니다. 일반적인 소스·설정·상태 파일은 무시 규칙의 기본 차단 대상이며, 강제로 추적하더라도 원격 변경 검사에서 거부됩니다.

## 분석

무시 규칙과 원격 검사를 함께 사용하면 단순 실수에 대한 이중 방어가 됩니다. 다만 Markdown 본문에 붙여 넣은 비밀 값이나 개인 정보는 확장자 검사로 탐지할 수 없습니다.

## 한계

내용 기반 비밀 탐지와 개인 정보 자동 비식별화는 이번 범위에 포함하지 않았습니다. 문서 게시 전 사람의 내용 검토가 계속 필요합니다.

## 결론

문서 형식 경계는 자동 검증할 수 있습니다. 공개 전 내용 검토를 필수 절차로 유지합니다.
