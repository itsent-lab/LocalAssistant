# 평가 기준: 제품 목표와 검증 지도

- 기준일: 2026-07-13
- 성격: 궁극적 제품 목표와 현재 검증 근거를 연결하는 판단 기준
- 코드 변경: 없음

## 목적

테스트 통과와 제품 완성은 같은 뜻이 아니다. 이 문서는 Local Assistant가 궁극적으로 제공하려는 사용자 경험, 이를 가능하게 할 구조 가설, 현재 근거, 아직 모르는 것과 다음 통과 조건을 연결한다.

## 한 문장 목표

> 사용자의 사적인 자료와 기억을 사용자가 소유한 환경에 유지하고, 자연어만으로 생활·업무·관심사와 생각의 흐름을 이어서 이해하며, 사용할수록 사용자의 유지보수 부담을 줄이는 개인 비서이자 사고 파트너

## 검증 수준

| 수준 | 의미 | 확인할 수 있는 것 | 아직 확인할 수 없는 것 |
| --- | --- | --- | --- |
| V0 구조 제안 | 책임과 경계를 문서로 검토 | 기술적으로 가능한가 | 실제 동작과 안전성 |
| V1 단위 검증 | 정책과 Adapter를 격리 검사 | 개별 계약이 지정 입력에서 맞는가 | 실제 흐름과 운영 장애 |
| V2 합성 종단 검증 | 개인정보 없는 fixture로 전체 흐름 검사 | 여러 계층이 함께 일반화되는가 | 실제 자료와 장기 사용 |
| V3 단일 사용자 운영 | 실제 앱과 생활 질문에서 반복 사용 | 현재 환경에서 유용하고 복구 가능한가 | 다른 사용자·기기·환경 |
| V4 제한 사용자 파일럿 | 다른 사용자가 준비된 범위에서 독립 사용 | 개발자 도움 없이 사용할 수 있는가 | 일반 제품·조직 확장 |
| V5 장기·이식 검증 | 시간 경과, 모델·기기 교체와 이전 검사 | 기억과 관계가 실제로 이어지는가 | 모든 환경의 보편성 |

현재 영역별 근거는 V1~V3에 걸쳐 있다. 모바일 연결과 조직 확장은 V0 제품 가설이다. 테스트 수가 증가해도 V4·V5가 자동으로 증명되지는 않는다.

## 먼저 검증하는 네 기반

### Model Adapter

모델 요청 형식이 바뀌어도 Query Interpreter, Tool, 기억과 평가 계약을 다시 만들지 않는 것이 목표다. 현재 공통 요청 경계와 모델 수명주기의 V1~V2 근거는 있으나 여러 모델·공급자 교체 뒤 동등성을 비교한 V5 근거는 없다.

### Data Adapter

개인별 데이터 구조를 공통 엔진에 고정하지 않고 출처·권한·단위·자료 부재를 지키는 것이 목표다. 읽기 전용 허용 목록, 개인 연결부 분리와 합성 두 번째 원천의 V2 근거가 있다.

### Memory Lifecycle

대화 로그를 무조건 기억으로 삼지 않고 사실·선호·상태·교정·추정을 다른 수명과 신뢰도로 관리하는 것이 목표다. 구조와 단위·합성 근거는 있지만 수개월의 변화와 모델 교체 뒤 의미 보존은 미검증이다.

### 개인정보 보호 회귀 평가

생활 실패를 재현하면서 개인 원자료와 식별 정보를 테스트·Git에 남기지 않는 것이 목표다. 알려진 패턴을 합성 fixture로 바꾸는 V2 근거는 있으나 모든 민감 정보를 탐지한다는 보장은 없다.

| 계속 관찰하되 잡음을 줄인다 | 원문 보존, 변경 근거, 관심 필터와 주제 병합 | V2~V3 | 전달·무시·교정 결과로 알림 유용성 측정 |
| 다른 사용자·조직에 공통 기반을 재사용한다 | 공통 계약과 개인 연결부 분리, 합성 원천 | 조직은 V0 | 개인 파일럿 뒤 조직형 합성 원천·역할 정책 별도 평가 |

## 현재 단계의 정확한 표현

> 개인 실사용 환경에서 사용자 소유 비서의 핵심 계약을 V1~V3 수준으로 검증하는 개념 실증이자 운영 프로토타입

확인된 것은 로컬 모델·사용자 소유 Wiki·허용 데이터와 Tool을 하나의 대화 경로로 연결하고, 개인 연결부를 공통 계약에서 분리하며, 사용자 교정과 자동 관찰을 구분하고, 실제 Mac 앱에서 운영 경로를 반복 실행할 수 있다는 점이다.

아직 다른 사용자의 독립 사용성, 수개월·수년 뒤 기억 정확성, 모델·기기 이전 뒤 연속성, 모바일 연결의 실제 보안과 조직 환경의 운영 가능성은 확인하지 않았다.

## 단계별 제품 검증 사다리

1. 기반 계약 안정화: 실제 실패를 합성 fixture로 바꾸고 수정 전 실패·수정 후 성공·유사 표현·기존 비회귀를 함께 검사한다.
2. 유지보수 부담이 줄어드는 개인 실사용: 반복 설명, 수동 Tool 수정, 기억 교정과 장애 개입이 감소하는지 측정한다.
3. 제한된 원격·다른 사용자 파일럿: 인증·권한·자료 부재와 오프라인 복구가 안전한 범위에서 독립 사용성을 확인한다.
4. 모델·기기 이식과 장기 기억: 교체와 이전 뒤에도 기억의 출처·의미·충돌, Tool 권한과 사용법이 유지되는지 확인한다.
5. 일반화 가능한 제품 기반: 개인 연결부를 수정하지 않고 새 비개인 원천과 환경별 권한·감사 정책을 연결한다.

## 앞으로 측정할 지표

- 자연어 과업 성공률과 사용자 개입률
- 명시적 정정 뒤 유사 실패 재발률
- 답변 근거·시각·단위 추적 가능률
- 자료 부재 시 추측하지 않는 비율
- 호출된 기억의 정확성과 현재성
- 전달된 알림의 유용성과 잡음 비율
- 장애 뒤 자동 복구율
- 개인정보 경계 위반 건수
- 모델·기기 이전 뒤 평가 유지율

현재 기준선이 없는 지표는 숫자를 추정하지 않는다. 충분한 실제 사례가 쌓인 뒤에만 통과 임계값을 정한다.
