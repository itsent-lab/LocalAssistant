# 개발 기록: 웹 감시 분류 알림과 메타데이터 정리

- 날짜: 2026-07-15
- 상태: 부분 완료

## 목표와 핵심 주제

웹 감시 결과를 수집 사이트별 알림이 아니라 사용자가 지정한 분류별 digest로 전달하고, 게시판 탐색 UI와 변동 숫자가 실제 새 자료처럼 보이지 않게 한다. 수집 원본과 식별 근거는 내부에 보존하되 사용자 표현은 제목과 판단에 필요한 핵심 변화에 집중한다.

## 문제

목록형 웹페이지에서는 게시 시각, 조회·댓글 수, 게시물 번호, 회원 집계와 페이지 탐색 카운터가 본문 행처럼 추출될 수 있었다. 숫자가 바뀌면 이전 행의 삭제와 새 행의 추가로 오인됐고, 생성 단계의 지시가 실패하면 불필요한 행이 최종 알림까지 남았다. 같은 분류의 감시도 서로 다른 예약 시각에 실행되면 여러 알림으로 나뉘었으며, 긴 페이지 평문이 한 목록 행에 섞이는 사례도 있었다.

## 진행 과정과 판단

처음에는 한 예약 슬롯 안의 무작위 실행 지연만 모으는 짧은 버퍼를 적용했다. 실제 운영 관찰에서는 서로 다른 기준 슬롯의 같은 분류에도 항목이 겹칠 수 있어 이 경계가 충분하지 않았다. 이후 자동 분류 알림을 기본 3시간 영속 digest로 바꾸고, 같은 전달 정책과 분류의 동일 항목을 한 번만 유지하도록 결정했다. 수동 즉시 확인과 서로 다른 전달 대상은 합치지 않는다.

표현 단계에서 특정 숫자나 실제 사이트 문구를 제외 목록으로 고정하지 않았다. 날짜·시각 뒤에 집계 숫자와 게시물 번호가 이어지는 게시판형 행, 탐색 카운터와 회원 배지를 공통 메타데이터 경계에서 판정해 snapshot 비교 전에 제거한다. 제목 뒤의 댓글 수와 새 글 표시는 제목을 보존한 채 suffix만 정규화한다. 생성 모델이 설명을 덧붙여도 최종 알림 경계에서 다시 정리하고 표시 건수를 재계산한다.

## 고민과 trade-off

알림을 즉시 보내면 변화는 빠르게 알 수 있지만, 같은 분류가 여러 수집 시각에 나뉘어 반복 전달될 수 있다. 기본 3시간 digest는 반복을 줄이고 읽기 쉬운 묶음을 만드는 대신 첫 자료 전달을 늦출 수 있다. 이 지연은 설정으로 되돌릴 수 있으며 수동 확인에는 적용하지 않는다.

모든 숫자 행을 삭제하면 숫자나 날짜로 시작하는 실제 제목까지 잃을 수 있다. 따라서 숫자 자체가 아니라 게시판 메타데이터의 구조적 조합을 판정하고, 실제 제목 문자가 이어지는 합성 인접 사례는 보존했다.

## 변경 범위

- 저장된 사용자 분류를 알림 제목과 병합 기준으로 보존한다.
- 같은 분류의 자동 결과를 영속 digest로 모으고 동일 항목을 중복 제거한다.
- 게시판 숫자 행과 페이지 탐색 UI를 snapshot 비교 전에 제외한다.
- 제목에 붙은 댓글 수·새 글 표시만 정규화하고 제목은 유지한다.
- 최종 알림을 일반 텍스트로 정리하고 과도하게 긴 단일 행을 제한한다.
- 이미 외부 채널이나 운영체제 알림 센터에 전달된 과거 표시는 소급 변경하지 않는다.

## 결과

사용자는 수집 원천 이름보다 자신이 지정한 분류를 기준으로 새 자료를 확인할 수 있다. 같은 분류의 인접 결과는 하나의 digest로 모이고, 조회 수 변화나 탐색 UI만 바뀐 경우에는 새 자료 근거가 되지 않는다. 기존 로컬 알림함도 조회 시 같은 최종 표현 정리를 적용한다.

## 완료 조건

수정 전에는 익명화한 날짜형·시각형·공지형 게시판 행이 변동 메타데이터로 판정되지 않고 최종 알림에도 남는 현상을 재현했다. 수정 후에는 세 유형을 제거하면서 날짜·숫자로 시작하는 실제 제목형 인접 사례를 보존했고, 동일 분류 병합·전달 대상 분리·수동 경로 유지·긴 행 제한을 회귀 테스트로 확인했다.

## 알려진 한계

한 운영 환경의 합성·부분 검증은 모든 게시판 HTML 구조의 일반화를 증명하지 않는다. 새로운 구조가 확인되면 실제 원문이나 출처를 공개 문서에 복사하지 않고, 개인정보 없는 합성 fixture로 실패 계층과 인접 보존 사례를 함께 추가해야 한다. 다음 자동 digest가 실제 운영에서 한 번만 전달되는지는 계속 관찰할 범위다.
