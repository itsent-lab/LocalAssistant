# 개발 기록: 항목별 자료 수집과 Wiki Browser 편집

- 날짜: 2026-07-13
- 상태: 코드·회귀 테스트·앱 빌드 검증 완료, 실행 앱 반영 전
- 관련 평가: [현재 작업분 검증](../evaluations/2026-07-13-working-tree-validation.md)

## 문제

구조화 게시판의 목록 전체를 하나의 문서로 저장하면 개별 글의 실제 출처, 내용과 갱신 이력을 파악하기 어렵습니다. 사이트마다 주제명이 달라 같은 분야의 알림이 여러 묶음으로 나뉘고, 첨부파일 다운로드 동작이 게시물처럼 수집될 가능성도 있었습니다.

Wiki Browser는 문서를 찾아 읽을 수 있었지만 사람이 직접 안전하게 수정하거나 AI 제안을 검토하고 관련 문서를 연결하는 흐름이 부족했습니다.

## 목표와 핵심 주제

목표는 많이 수집하는 감시 도구가 아니라, 사용자가 나중에 사람의 눈으로 읽고 판단하며 다시 활용할 수 있는 사용자 소유 지식 흐름을 만드는 것입니다.

- 목록과 알림보다 실제 항목의 원문과 출처를 보존합니다.
- 수집 원문, 검토된 지식과 사용자가 직접 수정한 문서를 구분합니다.
- AI는 번역·교정·연결을 돕지만 원문을 자동으로 확정하거나 사용자 수정을 덮어쓰지 않습니다.
- 사이트와 모델이 바뀌어도 출처, 분류, 시각과 판단 근거가 Markdown에 남게 합니다.

성공 기준은 수집 건수가 아니라 개별 자료의 출처성과 검토 가능성, 중복 없는 갱신, 실패 시 원문 보존과 사용자 최종 통제권입니다.

## 진행 과정과 판단

처음에는 목록 페이지의 변경을 감지하고 목록 문서를 갱신했습니다. 실제 사용에서는 무엇이 바뀌었는지는 보여도 각 글의 본문과 출처 이력을 독립적으로 재사용하기 어려웠습니다. 목록을 발견용 index로만 사용하고 실제 게시물을 저장 단위로 바꾸는 방향을 선택했습니다.

모든 링크를 수집하면 누락은 줄지만 첨부파일과 다운로드 동작까지 문서가 되어 잡음이 커집니다. 파일·동작 전용 링크는 제외하되 다운로드를 설명하는 실제 게시물은 유지하는 경계를 회귀 사례로 고정했습니다.

출처별 주제명을 유지하면 출처는 잘 보이지만 같은 분야의 알림이 분산됩니다. 출처는 각 문서의 provenance에 남기고 폴더와 알림은 안정된 주제로 합쳤습니다.

영문 자료를 모두 번역하면 읽기는 편하지만 모델 지연이나 실패가 수집을 막을 수 있습니다. 번역은 선택 단계로 두고 실패하면 원문으로 계속 저장하도록 했습니다.

Wiki 문서를 AI가 즉시 수정하면 편리하지만 사용자 문장과 출처를 조용히 바꿀 위험이 있습니다. AI 결과는 제안으로만 보여주고 사용자가 적용한 뒤 별도로 저장하는 흐름을 선택했습니다.

## 남은 고민

- 주제 체계가 경직되지 않으면서도 폴더가 다시 파편화되지 않게 하는 기준
- 수집 원문을 언제 검토된 재사용 지식으로 승격할지에 대한 승인 흐름
- 로컬 모델 번역 품질과 처리 지연의 허용 범위
- 기존 자료 마이그레이션의 실제 대상이 생겼을 때 archive·복구 승인 단위

이 항목은 완료된 기능이 아니며 실제 사용 결과와 실패 사례가 쌓일 때 다시 판단합니다.

## 변경 범위

### 항목별 자료 수집

- 새 게시물과 제목이 바뀐 게시물의 상세 URL을 읽어 각각 독립된 문서로 저장합니다.
- 원문 URL, 제목, 게시 시각, 관찰 시각, 상위 감시 출처와 수집 이력을 보존합니다.
- 같은 원문 URL은 기존 문서를 갱신해 제목·분류 변화로 중복 문서를 만들지 않습니다.
- 파일과 다운로드 동작 링크를 실제 게시물 수집 대상에서 제외합니다.
- 수집 원문은 보존하지만 검토 없이 재사용 지식으로 승격하지 않습니다.
- 주제명을 안정된 2단계 분류로 정규화하고 여러 출처의 같은 주제를 하나의 알림으로 합칩니다.
- 영문 비중이 높은 자료는 로컬 모델이 한국어 제목과 한 문장 요약을 제안합니다. 번역에 실패해도 원문 수집은 계속합니다.
- 의미 있는 변경 근거가 없으면 기존 감시 기준선을 유지합니다.

### 기존 문서 마이그레이션

과거 목록 집계 문서를 항목별 문서로 바꾸는 전용 절차를 추가했습니다. 기본 동작은 변경 없는 사전 점검이며, 명시적으로 적용할 때만 상세 자료 수집, 항목별 문서 생성, 과거 집계 문서 보관, 색인과 상태 갱신을 수행합니다.

### Wiki Browser

- 지식, 전체, 수집 범위를 나눠 지식 문서와 수집 원문을 구분합니다.
- Markdown을 읽기 좋은 형식으로 표시합니다.
- 읽기와 편집 모드를 분리하고 저장하지 않은 수정이 있을 때 이동 전 확인합니다.
- 문서를 읽은 시점의 revision을 저장 시 비교해 외부 변경을 덮어쓰지 않습니다.
- AI 교정은 자동 저장하지 않고 변경 미리보기를 검토한 뒤 편집기에 적용하도록 합니다.
- 긴 문서를 잘라서 불완전하게 교정하지 않도록 24,000자를 넘으면 요청을 거절합니다.
- 관련 문서 후보를 검색한 뒤 로컬 모델이 실제 관련성을 검토합니다.

## 결과

수집 자료의 기본 단위가 목록 페이지에서 실제 게시물로 바뀌어 출처와 갱신 이력을 항목별로 확인할 수 있습니다. 사용자는 사이트별 알림 대신 같은 주제의 새 자료를 한 번에 검토할 수 있습니다.

Wiki Browser는 검색·열람 창에서 사람 중심의 문서 편집 도구로 확장됐습니다. AI는 직접 원문을 바꾸지 않고 교정과 연결 후보를 제안하며, 최종 적용과 저장은 사용자가 통제합니다.
