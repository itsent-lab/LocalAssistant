# 문서 작성 지침

이 저장소의 변경은 문서에만 한정합니다. 새 기록은 아래 순서로 작성합니다.

1. 실제 구현 또는 수정이 끝났는지 확인합니다.
2. 개인 정보와 내부 운영 정보를 제거합니다.
3. `docs/templates/`의 알맞은 양식을 복사합니다.
4. 목표, 진행 과정, 판단 근거, 변경 범위, 검증 수치, 결과와 알려진 한계를 작성합니다.
5. 문서 전용 검사가 통과하는지 확인합니다.

## 기록할 내용

- 만들려는 제품의 목표, 핵심 주제와 해결하려는 문제
- 중요하게 여기는 가치, 성공 기준과 의도적으로 제외한 범위
- 주요 가설, 시도, 실패, 예상 밖 결과와 방향 전환 계기
- 검토한 대안, 장단점, trade-off와 선택·보류 이유
- 실제 실행한 테스트의 종류·범위, 총 개수, 통과, 실패, 오류와 건너뜀 숫자
- 부분 테스트나 미실행 사유, 알려진 한계와 남은 검증

대화 원문이나 내부 추론을 그대로 옮기지 않습니다. 공개 가능한 근거를 바탕으로 다음 개발 판단에 재사용할 수 있는 과정과 결정 이유로 정리합니다.

## 금지 항목

- 애플리케이션 소스 코드와 테스트 코드
- 실행 가능한 셸·Python·JavaScript 등의 스크립트
- API 키, 토큰, 계정, 내부 URL과 인증 정보
- 개인 데이터, 가족 데이터와 식별 가능한 대화 원문
- 로컬 절대 경로, 데이터베이스 접속 정보와 상태 파일
- 빌드 산출물, 로그 원문, 캐시와 바이너리

명령이나 코드 조각은 재현에 꼭 필요한 짧은 예시만 문서 안에 설명용으로 포함합니다. 구현을 복원할 수 있는 전체 소스는 포함하지 않습니다.
