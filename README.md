# LocalAssistant 개발 문서

이 저장소는 LocalAssistant의 **개발 과정과 검증 결과를 공개하기 위한 문서 전용 저장소**입니다.

애플리케이션 소스 코드, 실행 설정, 개인 데이터, 인증 정보와 로컬 운영 상태는 이 저장소에 포함하지 않습니다. 문서는 실제로 구현하거나 검증한 내용만 기록하며, 계획이나 가설은 현재 동작과 명확히 구분합니다.

개발 리포트의 원문은 비공개 소스 저장소에도 함께 보존합니다. 이 GitHub 저장소는 그중 공개 정책을 통과한 문서만 단방향으로 게시하는 공개 문서 저장소이며, 소스 저장소를 미러링하지 않습니다.

## 세 가지 핵심 사상

### 사용자 소유의 연속성

모델은 교체 가능한 추론 부품입니다. 사용자 소유 기억, 교정, 결정 이유와 Tool 계약을 모델 밖에 유지해 기기·모델·공급자가 바뀌어도 개인화를 이어갑니다.

### 모델 밖에서 누적되는 다층 개인화

사실, 지속 선호, 일시 상태, 사용자 교정, Routing 경험과 Turn Trace를 서로 다른 수명·신뢰도·권한 계층으로 관리합니다. 현재 차별점은 출처가 있는 기억 수명주기와 결정적 권한·Tool 검증입니다.

기분·피로·인지 부하와 시간 맥락을 만료 가능한 상태로 다루는 `Dynamic State Vector`는 연구 방향입니다. 실제 벡터 갱신·감쇠·사용자 제어가 구현되기 전에는 현재 기능으로 주장하지 않습니다.

### 1,000줄 책임 모듈 상한

모든 신규 책임 모듈은 주석과 빈 줄을 포함해 1,000줄 이하로 제한하고, 책임이 둘로 갈리면 줄 수와 무관하게 분리합니다. 공개 호환성과 실행 조립을 유지하는 composition root도 현재 962줄이며 새 비즈니스 로직과 순증가를 허용하지 않습니다. 이는 극단적 경량화 주장이 아니라 god-file 재발을 막는 유지보수 계약입니다.

## 원칙

1. 소스 코드와 실행 가능한 스크립트를 게시하지 않습니다.
2. 개인·가족 자료, 로컬 경로, 비밀 값과 원문 대화는 게시하지 않습니다.
3. 수정 내용에는 검증 방법과 관찰된 결과를 함께 기록합니다.
4. 실패와 한계도 성공 결과와 같은 수준으로 남깁니다.
5. 미래 계획은 완료된 기능처럼 표현하지 않습니다.
